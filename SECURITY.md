# Security Policy

## Alcance

Este repositorio contiene documentación técnica, scripts de auditoría y configuración para construir un sitio estático con MkDocs.

No debe contener:

- credenciales;
- tokens de GitHub;
- datos personales;
- datos de pacientes;
- información propietaria de aseguradoras, EPS, IPS, proveedores o administradores;
- bases de datos reales de reclamaciones;
- archivos con información protegida por confidencialidad.

## Reporte de problemas

Si encuentras un problema de seguridad o exposición accidental de información sensible:

1. No abras un issue público con detalles sensibles.
2. Contacta directamente al mantenedor del repositorio.
3. Incluye:
   - archivo afectado;
   - tipo de información expuesta;
   - impacto potencial;
   - recomendación de remediación, si aplica.

## Buenas prácticas para contribuciones

- Usa datos sintéticos en ejemplos.
- Revisa archivos antes de hacer commit.
- No subas notebooks, CSV, Excel, PDFs o capturas con datos reales sin revisión.
- Evita incluir rutas locales personales en documentación final.
- No publiques backups generados localmente.
